# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A French-language toolkit for learning modern Hebrew, deployed as static files on GitHub Pages at `https://rubischtgadol.github.io/flashcards-hebreu/`. There are **no dependencies, no tests, no package manager** — every deployed file is a single self-contained HTML document with inline CSS and vanilla JS. The only tooling is `build.js` and `verifie_exemples.js` (dev-only, zero-dependency Node scripts, not deployed). To develop, open the files in a browser; to deploy, replace files on the `main` branch (GitHub Pages redeploys automatically, same URLs).

Important: `app.html` uses `fetch()`, so it must be served over HTTP, not opened as a `file://`. Run a local server from the repo root (e.g. `python3 -m http.server`) and open `http://localhost:8000/`.

Verification tooling in this WSL environment: visual checks run through **Playwright + real WebKit** (Safari's engine, matching the target iPhone — `devices['iPhone 16 Pro']` for mobile), pure logic through jsdom; the system headless Chrome hangs in WSL2 — never use it. Browsers persist in `~/.cache/ms-playwright`; only the npm `playwright` lib needs reinstalling per session (in the scratchpad). Operational details in TODO.md § Outillage.

## The files

- **`vocabulaire_hebreu.html`** — the grammar + vocabulary notebook. This is the **single source of truth** for all content. Editing vocabulary means editing this file. Since 2026-07-19 it also carries the a11y layer the app already had: **every Hebrew node has `lang="he"`** (100% of 5015 nodes as of 2026-07-19 — pure-Hebrew spans carry the attribute directly, Hebrew runs inside French prose are wrapped in `<span lang="he">`; keep this when adding content. The count must be **measured in a browser, not computed from the source**: adding one entry also creates its generated `span.cursive`, so a new word is worth more than one node), plus a `prefers-reduced-motion` guard (which must include `scroll-behavior:auto` — the global `transition:none` rule does *not* cover smooth scrolling), a `@media (pointer:coarse)` block holding touch targets at 44px, and (since 2026-07-19) the same global gold `:focus-visible` ring as the other two files. Its CSS declares **two named micro-title voices** — `thead th, .subtheme` (Assistant 700 / 0.84rem / 0.12em / gold) and `.toc-group-label, .part .part-num` (JetBrains Mono / 0.7rem / 0.14em) — which replaced four ad-hoc specs; put a new label on one of the two rather than inventing a third. Since 2026-07-19 it also carries an **8-step type ramp** in a *second* `:root` block (`--pas-titre` 2.3rem … `--pas-micro` 0.7rem, see DESIGN.md §3) that replaced 24 drifting values; the first `:root` stays the byte-identical shared token block, so never merge the two. **No literal font-size may remain outside the ramp** — the single named exception is `1.15em` on Hebrew embedded in French prose ("one notch above whatever surrounds me"). ⚠️ **`font-size:22px` sits on `body`, never on `html`, in all three files — so `1rem` is 16px, not 22px** (measured in WebKit; the old comment claiming otherwise was false, and that misunderstanding is what caused the drift). Do not "fix" this by moving the 22px to `html`: it would scale every `rem` by 1.375 here *and* in `app.html`, whose sizes are tuned to what they actually render. Hebrew inside French prose is rescued by `span[lang="he"]:not([class])` (serif + 1.15em), with `thead th` and `.tr` explicitly opted out so their declared voices keep the upper hand. Three conventions worth not breaking: `border:1px dashed` means **only** "nothing here" (`.empty`; warning boxes use the `.attention` class, solid rule), **no surface is gold-tinted at rest** (both `.part` and `.tip` were quieted — only the app's « Révision du jour » card keeps that licence), and content sits inside a `<main>` landmark while the search bar and the `<nav>` table of contents stay outside it. Note `build.js`'s `firstSpanText` matches `class="he"` exactly and tolerates other attributes on the span, so `lang` is safe to add; verify with `node build.js --check` regardless.
- **`index.html`** — the **portal** (the root page), in two stages: a full-screen welcome — the personalized greeting « Ruben vous souhaite la bienvenue ! » or its Hebrew version « ראובן מקבל אתכם בברכה! » (the idiomatic לקבל בברכה construction for « souhaiter la bienvenue » — random draw, no nikud, the first name deliberately public, an acknowledged exception to the repo-anonymization decision), the icon's golden א as a vector glyph below it, and two 7-branch menorah SVGs (inline `<symbol>`, glowing flames) lighting the sides; one tap/click to enter (the welcome overlay only exists when JS runs; without JS the doors show directly) — then the choice between two **strictly equal** doors (flashcards app, notebook — neither is gold-highlighted at rest). Same `:root` tokens as the rest; it registers the service worker too. No vocabulary, no build coupling — safe to edit freely.
- **`app.html`** — the online flashcards app. It does **not** contain vocabulary; at load it fetches `vocabulaire_hebreu.html` and extracts the cards at runtime. It is also the **source of the standalone file**: the network-only code (fetch + runtime extraction) is fenced by `BUILD:ONLINE-ONLY` marker comments that `build.js` relies on.
- **`flashcards_hebreu.html`** — the standalone offline flashcards. **100% generated by `build.js` — never edit it by hand.** It is `app.html` with the `BUILD:ONLINE-ONLY` block replaced by an embedded `const CARDS = [...]` snapshot and a direct boot (no loader).
- **`build.js`** — regenerates `flashcards_hebreu.html` from the two files above, printing the per-section card count and failing if an expected section drops to 0, if an `EXPECTED_LEVELS` level drops to 0, or if **any card lacks `data-niveau`** (level-coverage guard, naming the offending words). `node build.js --check` verifies without writing. **Run it after any edit to the notebook or to `app.html`.** It re-implements `extractCards()` with regex parsing (no DOM in Node) — any change to `extractCards()` in `app.html` must be mirrored there, and `--check` byte-compares so drift is caught.
- **`verifie_exemples.js`** — dev-only companion of `build.js` (zero-dependency): validates every usage example in the notebook (fields, 3–8 Hebrew words, nikud on every word, `.tr` consistent with `he2tr`/`trKey` extracted from `app.html`, sentence vocabulary present in the notebook lexicon and **at most one CEFR level above** the word's — it warns only at +2, since a natural everyday sentence for an A1 verb needs concrete A2 nouns). Its lexicon is built from the cards **and** from the Hebrew of the grammar sections (which produce no cards, yet teach forms like שֶׁלְּךָ); two guards keep that from backfiring — `<ul class="exemples">` are excluded (otherwise each sentence would validate its own vocabulary) and a grammar word may only *add* an unknown entry at level 0, never lower a known card's level (`feed()` keeps the **lowest** level, so reusing it there would silently disable the level check). See ARCHITECTURE.md §5.1. It also enforces the **coverage rule** (2026-07-19 decision): every entry of the Noms/Adjectifs/Verbes tables must carry at least one example (verbs: a present-tense sentence) — a word added to those tables without its `<ul class="exemples">` is a blocking error. **Run it after adding or editing vocabulary or examples**; errors block, warnings are editorial signals (`--strict` to block on them too).
- **`manifest.webmanifest`, `sw.js`, `icons/`** — the PWA layer (install + offline). The service worker registration lives **inside** the `BUILD:ONLINE-ONLY` block of `app.html` and must stay there (the standalone file must not register it); the portal has its own small registration snippet. In `sw.js`, only **root** navigations are collapsed onto the `./` shell — which is now the **portal**; deep pages (`app.html`, the notebook) must be served as-is. `start_url` in the manifest points to `./` (the installed icon opens the **portal** — a 2026-07-19 decision reversing the previous `./app.html`). Bump `VERSION` in `sw.js` after changing its strategy, the asset list, or the icons (v8 = two-stage portal welcome + start_url back to the portal). The `:root` design-token block must stay **identical** between the notebook, `app.html` and the portal (notebook is the reference); if `--bg`/`--gold` change, regenerate the icons and update `manifest.webmanifest`/`theme-color`.

## The extraction coupling (most important thing to understand)

`app.html`'s `extractCards(doc)` (near line 2086) scrapes the notebook's DOM. Its parsing is tightly coupled to the notebook's markup, so edits to `vocabulaire_hebreu.html` can silently drop cards if they break these assumptions:

- Sections are found by `<h2>` elements containing a `<span class="count">LABEL</span>`. The **exact `LABEL` text** is the key used throughout `extractCards` (e.g. `'Verbes'`, `'Adjectifs'`, `'Noms'`, `'Pronoms personnels'`, `'Nombres (0–10)'`). Renaming a count label detaches its section from extraction.
- Word entries live in either `<table><tbody><tr>` rows (Verbes, Adjectifs, Noms) or `<ul class="word-list"><li>` items (pronouns, prepositions, numbers, expressions, **Phrases**, etc. — see the `listCats` map). The `Phrases` section holds full example sentences (cat `Phrases`); they flow through every mode as ordinary cards and get a smaller `.big-he.phrase`/`.big-fr.phrase` size so long sentences wrap cleanly. Adding a `listCats` entry must be mirrored in `build.js` (both the `listCats` object **and** `EXPECTED_CATS`).
- Cells/items expose fields via child spans: `.he` (Hebrew w/ nikud), `.tr` (transliteration), `.fr` (French). Table extraction is **positional** — it reads fixed column counts and indexes (Verbes needs ≥5 columns → il/elle/ils/elles forms; Adjectifs ≥4; Noms ≥3 where column 2 is the gender `m`/`f` and column 3 the plural).
- A `<li>` can carry `data-fr-court` (short French used on the card instead of the notebook's long `.fr`) and `data-note` (precision shown under the answer). This is how card-specific wording lives in the notebook rather than in app code.
- Every `<li>` **and** every `<tr>` of the three tables can carry `data-niveau="A1"…"C2"` (fine-grained CEFR level, classification method in ARCHITECTURE.md §4). Optional *in the app* by design: an unclassified word ignores the level filter and stays visible everywhere. But it is **mandatory in the notebook** — `build.js` prints per-level counts, fails if a level in `EXPECTED_LEVELS` drops to 0, **and fails naming the word if any card comes out without a `niveau` at all** (coverage guard added 2026-07-19, modelled on `verifie_exemples.js`'s example-coverage rule; a « couverture N/N » line reports what it measured). Coverage has been 713/713 since 2026-07-19. The app's tolerance stays as a safety net, not as a licence to skip the attribute.
- A word can carry usage examples: a nested `<ul class="exemples"><li>` (spans `.he`/`.tr`/`.fr`) inside the word's `<li>`, or at the **end** of the first `<td>` for table rows (order matters — extraction reads the *first* `.he`/`.fr` of the fragment, so the word's own spans must come first). ⚠️ Nested `<li>` break naive regex: `lisOf` in build.js does depth-aware scanning for top-level `<li>` only, and app.html uses the `ul.word-list > li` direct-child selector — keep both properties when touching the parsers. Property insertion order in card objects must stay identical between the two extractors (`--check` byte-compares the embedded snapshot).
- Grammar-only sections (racine, passé, futur, binyanim, article, smikhut…) have no `.count` mapping into cards and are intentionally excluded from flashcards.

The card object schema produced (and embedded in the standalone file) is:
`{cat, he, tr, fr, note?, niveau?, exemples?:[{he, tr, fr, he_plain}], he_plain, forms?:[{he, tr, label, he_plain}], genre?}` where `he_plain` is `he` with nikud stripped (`stripNikud`) and `niveau` is the raw CEFR value from `data-niveau` (absent if unclassified). Table-derived cards have `tr:''`; the UI falls back to `he2tr(card.he)` for display.

Two guards catch silent extraction breakage: `init()` in `app.html` warns (console + setup screen) when an expected category yields 0 cards, and `node build.js` prints per-section counts and exits non-zero on an empty section. After editing the notebook: run `node build.js`, check the counts, and verify in the browser that the loader shows the expected "N mots chargés".

## Answer checking (the flashcards' hardest logic)

`checkAnswer` (near line 1761) grades typed answers leniently and returns `'exact'`, `'almost'`, or `false` — truthy means accepted, and `'almost'` (accepted only via the `editDist` tolerance) makes `showInputFeedback` display the « ✓ Presque ! La forme exacte : » verdict (green, attempt shown un-struck). Feedback kinds are `'ok' | 'almost' | 'no' | 'skip'`; `fixVerdict` treats `ok`/`almost` as "was counted good". Details:

- **French direction** (`he2fr`): `normFr` strips accents/case, and `frVariants` splits the French field on `/`, `,`, parentheses, and articles so multiple phrasings pass.
- **Hebrew direction** (`fr2he`): accepts either real Hebrew (typed via the built-in virtual Israeli-layout keyboard, rows defined near line 1906 — folded behind the « Clavier hébreu » toggle, and **hidden entirely on touch devices** via `@media (pointer:coarse)` with `!important`, a 2026-07-19 decision: iOS provides a native Hebrew keyboard, the virtual one only serves physical AZERTY desktop keyboards) **or** loose French-style transliteration. Transliteration is normalized to a canonical key by `trKey` (folds `ph→f`, `kh/ch→h`, `q→k`, `w→v`, `tz/ts`, `ou→u`, strips apostrophes, collapses doubles) and compared against `he2tr(card.he)` — a nikud-driven Hebrew→transliteration generator — with a small Levenshtein (`editDist`) tolerance. When changing acceptance behavior, edit `trKey`/`he2tr` together since they must agree on the canonical form — and note `he2tr` is also used for **display** wherever a card has no notebook `tr`.

**Transliteration standard** (all notebook `.tr` values and `he2tr` output follow it): `kh` = khaf without dagesh, `ch` = het (final furtive patach → `ach`), `ts` = tsadi, `'` = ayin everywhere (`'ivrit`, `be'er`, `yode'a`) and alef between two vowels (`tsme'ah`), final hey kept (`atah`, `zeh`), `ei` for tsere/segol+yud (`beit`), `u`/`f`/`k`/`v` (never `ou`/`ph`/`q`/`w`). Initial cluster shva is written only when heard (`gdolim` but `ledaber`) — the notebook is authoritative there; don't blindly regenerate hand-written `.tr` values from `he2tr`. Thanks to `trKey`'s folding, the standard is display-only: typing stays tolerant to all variants.

## UI settings model

The setup screen opens on the « Révision du jour » card (the daily action leads); the search drawer sits right below the mastery bar, before the categories. It uses `.chip` segmented toggles carrying `data-*` attributes (`data-mode`, `data-dir`, `data-script`, `data-order`, `data-audio`, `data-len`), wired in a loop over `SEG_KEYS` by `segPick(container, key, btn)`. Every `.seg` (and `#cats`) is a `role="group"` labelled by its `<h2>` (`aria-labelledby`, notes via `aria-describedby`). **Three native `<details class="adv">` folds** share one visual row: the set-once groups (Ordre/Longueur/Prononciation) in `#adv`, plus `#fold-cats` and `#fold-niv` (2026-07-19) which fold the panel's two biggest decision points — 1278px → 874px, 43 → 35 tab stops. In a fold, the group's `<h2>` **is** the `<summary>`'s row (it stays the `aria-labelledby` target, so the accessible name is not doubled) and therefore takes the fold-label voice, not the gold Title voice — a folded group reads as a fold, an unfolded one as a section. Each `<summary>` carries a `.adv-sub` summary of the current selection (`refreshFoldSubs()`, called from `updateStart()` so every selection path refreshes it; ≥3 entries are counted rather than listed, since an ellipsis-clipped list would lie; order follows `catOrder`, i.e. the on-screen chip order). ⚠️ Open/closed is decided **at load only** (`applyFoldState()`, from `applyPrefs()`): open while the selection is empty — otherwise a blank profile, which by decision has nothing selected, would offer nothing to do and `#start-hint` would point at invisible chips. After that the fold obeys only the user; snapping it shut under their finger on the first pick would be hostile. `buildNivChips()` hides the whole `#fold-niv` when the notebook has no `data-niveau` at all. under `pointer:coarse` the `#start` button is `position:sticky` at the bottom **only while enabled** (`.start:enabled` — a 2026-07-19 critique fix: sticky *and* disabled, it floated a translucent gold slab over four category chips and ate their taps, `elementFromPoint` returning `start`). `#start:disabled` therefore carries a full opaque skin (`background:none`, `--line` border, `--ink-dim` text) instead of the old `opacity:.4` on the gold gradient — gold must never appear on an inactive state (DESIGN.md, règle de la lampe), and an opacity let page content show through the button. The empty-selection hint `#start-hint` sits **above** it in the DOM so it stays visible while stuck, and carries `role="status"` so its three messages are announced. Modes (`state.mode`): `cards` (flip), `input` (typed), `quiz` (QCM — 4 choices, distractors from `pickDistractors`, same category first, **rejecting any candidate whose French variants are within Levenshtein 1 of already-kept options** so choices are never quasi-synonyms; a relaxed last-resort pass still guarantees 4 options). Each mode has its own zone in the study screen (`#controls-cards` / `#input-zone` / `#quiz-zone`) and a `setup*Card()` that toggles the `input-mode`/`quiz-mode` body classes; `render()` branches on `state.mode`. Direction he→fr / fr→he; script nikud / plain / cursive. Audio uses the browser's Hebrew SpeechSynthesis voice: `'click'` means **only** the speaker button triggers speech (all answer paths honor this; the **P key** counts as a deliberate trigger and follows the speaker button's visibility rules), `'auto'` speaks on render/reveal; with no Hebrew voice, `reflectVoiceUi()` sets `body.no-he-voice` (speaker buttons + `.spk-hint` hidden) **and** disables the `#audio` chips; when a voice is found, `#audio-note` displays **its real name and, on a second `.voice-id` line (mono, LTR), its `voiceURI`** (« Voix hébraïque détectée ✓ — Carmit » / « identifiant : com.apple.ttsbundle.Carmit-compact ») — the diagnostic for the robotic-voice issue. ⚠️ **Platform ceiling, established 2026-07-19**: on iOS the voice can never be an "Enhanced"/"Premium" variant — WebKit filters `getVoices()` down to `AVSpeechSynthesisVoiceQualityDefault` (WebKit bug 203689), and `name` never carries the quality anyway (it's a separate `AVSpeechSynthesisVoice` field the Web Speech API does not expose). Never advise installing a higher-quality system voice as a fix. **`voiceURI` is the only field that distinguishes quality, which is why it is now on screen**: a `…-compact` suffix confirms the ceiling, a `.enhanced.`/`.premium.` one would mean WebKit's filter changed and would reopen the question. See TODO.md point 3. Keyboard parity: cards Space/←/→, input Enter, quiz **1–4 to pick + Enter/Space for next** (a focused button keeps native handling), and **C** presses the current mode's correction button (« Corriger » / « Annuler la dernière réponse ») when it is visible and enabled — both P and C ignore Ctrl/Cmd/Alt combos; `#kbd-hint` content adapts per mode. The done screen lists missed cards in `#missed-list` (populated in `finish()`, rows `.m-he`/`.m-fr`); `#retry-all` is relabelled "Rejouer ces N cartes" from `origQueue.length`, and a review-session finish with misses appends the "ratées aussitôt redevenues dues" explanation to `#done-msg`.

**Levels filter**: the « Niveau » group (`#niv`, multi-select chips like `#cats`, built by `buildNivChips` from the cards' `niveau` field) folds fine CEFR into four labels via the `NIVEAUX` table (Facile = A1, Intermédiaire = A2–B1, Difficile = B2–C1, Expert = C2). Only non-empty levels render a chip (no C2 in the notebook yet → no « Expert » chip); a notebook with no `data-niveau` at all hides the whole group. `start()` filters the pool with `nivOk(card)` (cats × niveau intersection; unclassified cards always pass); `startReview()` deliberately does **not** filter (a learned card stays due). `updateStart()` distinguishes three empty cases (no category / no level / empty intersection) with a dedicated `#start-hint` message each. Selection persists as `prefs_v1.niveaux` (backward compatible: missing or empty → all selected).

**Usage examples**: `exHtml(card)` renders the « Voir un exemple » disclosure (ghost `.ex-toggle` + `.ex-box`, label flipping to « Masquer l'exemple » while open, one `.ex-item` per example with a per-sentence Écouter button hidden under `no-he-voice`); it appears **only where the answer is already visible** — cards verso (`state.backHtml`), input feedback (`showInputFeedback`), quiz verdict (`#quiz-ex`, filled in `quizPick`, cleared in `setupQuizCard`) — never on the fr→he front (the example contains the word). Event wiring is `exBind` delegation (bindTap-style pointerup+click with `stopPropagation`, otherwise tapping the fold would flip the card); bound once on `#face-content`/`#feedback`/`#quiz-ex`/`#search-results`. The search drawer shows examples too (`.srd-ex`, and `sr-detail` expandability includes `c.exemples`).

**Session length** (`state.len`: `'10'|'20'|'50'|'all'`, default `'20'`): the « Longueur » group truncates the deck via `limitPool()`, applied *after* the shuffle in `start()` (random order = a different draw each session; linear order = the first N). `startReview()` applies it too, after sorting due cards most-overdue-first — the remainder stays due and reappears on the review card (whose subtitle announces the batch size when due > limit). "Rejouer les ratées" is deliberately **never** limited. `state.origQueue` holds the truncated deck, so "Recommencer" replays the same cards.

**Spaced repetition (Leitner)**: `recordResult(card, good)` is called from *every* answer path (cards `answer`, input `submitAnswer`/`skipAnswer`, `quizPick`) and persists a per-card box/due record to `localStorage` under `srs_v1` (card identity = `cat|he_plain`). The setup screen's "Révision du jour" button (`startReview`) builds a session from `dueCards()`; `refreshSrsUi()` (called at the end of `buildChips()`, so it runs in both the online and standalone boot paths) updates the due count and the mastery bar. Intervals in `SRS_INTERVALS`. This layer is invisible to `build.js` — it's pure app state.

**Persistence & resume** (also invisible to `build.js`, also restored from `buildChips()` → both boot paths): `applyPrefs()` restores `{cats, mode, dir, script, order, audio, len}` from `localStorage` key `prefs_v1` (saved by `savePrefs()` on every setting change) and reflects it into the chips' `aria-pressed`; first launch (no prefs) selects **no categories and no levels** — the user must choose (a 2026-07-19 decision replacing the old `defaultCats()` = everything-except-`Phrases` default); `#start` stays disabled and `updateStart()`'s `#start-hint` guides; the other six seg keys keep their initial values. Level backward compat: saved prefs *without* a `niveaux` field (pre-filter profiles) still select all levels; a present-but-empty array stays empty. `sessRestore()` rebuilds an in-progress session from `sessionStorage` key `sess_v1` (queue/missed/orig as card-id arrays + idx/score) and reopens `#study`; it drops the snapshot if the vocab changed underneath (missing id / idx out of range). `sessSave()` fires on each `render()` and answer; `sessClear()` on `finish`/`exit`/`back-setup`. **The verdict is reversible in all three modes**: `recordResult` snapshots the pre-write SRS entry (`lastRecord`); input mode's "Corriger" button (`fixVerdict`), quiz mode's `#quiz-fix` (`quizFixVerdict`, freezes into a confirmation label), and cards mode's "‹ Annuler la dernière réponse" (`undoCardAnswer`, using the `cardsUndo` snapshot to also restore `goodCount`/`missed`/`idx` and re-render the previous card) all go through `undoLastRecord`; `beginSession` resets both snapshots. Submitting an **empty input field is a no-op** (no miss counted, no SRS write). Exiting mid-session shows `#exit-note` ("Session interrompue — X/Y déjà comptées dans ta révision", `role="status"`) on the setup screen when ≥1 answer was given; hidden again by `beginSession`. The loader's error screen (`showLoaderError`, inside `BUILD:ONLINE-ONLY`) distinguishes `file://` / offline / unavailable and offers "Réessayer". **Profile reset** (« Repartir de zéro », bottom of the `#adv` fold): `#reset-btn` opens a two-step confirmation (`#reset-confirm`, cancel is the focused safe default, the warning names `masteryStats().seen`); confirming clears `srs_v1`/`prefs_v1`/`sess_v1`, resets in-memory `SRS`/`lastRecord`/`cardsUndo` **and the six `state` seg keys to their initial values** (`applyPrefs()` without stored prefs doesn't touch them), then refreshes in place (`applyPrefs`/`refreshSrsUi`/`updateStart`, `#exit-note` hidden) — no page reload; `#reset-done` (`role="status"`) announces completion.

**Accessibility invariants** (from the a11y passes): every generated Hebrew node carries `lang="he"` (`.big-he`, `.sub-he`, `.cursive-line`, `.f-he`, `.qc-he`, `.sr-he`, `.srd-he`, `.m-he`, feedback `.he`, brand `.he`) — keep it when touching those template strings. Focus is a single global `:focus-visible` gold outline (no bare `outline:none`). ⚠️ **Never use `transition:all` in `app.html`** — the shorthand captures the `outline-*` longhands and WebKit then pins them to their initial values (`medium`/`currentColor`/offset 0), so the focused element renders WebKit's UA ring instead of the gold one *while still matching `:focus-visible`*. This silently cost the 40+ chips and the cards-mode buttons their focus ring until 2026-07-19. Always list the animated properties explicitly (`background, color, border-color, opacity`). Live announcements: `#feedback` (`aria-live`), `#quiz-live` (visually-hidden QCM verdict, set in `quizPick`/`quizFixVerdict`, cleared in `setupQuizCard`), `#flip-live` (visually-hidden cards-mode verso — French + transliteration, plus an "exemple disponible" mention when the card has examples, set in `doFlip()`, cleared on unflip and each new card), `#done-msg`, `#loader-msg` and `#exit-note` (`role="status"`). `.bar` is a `role="progressbar"` updated in `render()`/`finish()`. Search rows are `role="button"` + `tabindex="0"` + keydown (a real `<button>` is impossible: the speak button is nested inside), with `aria-expanded` on expandable rows — one action per gesture: an expandable row only folds/unfolds (audio stays on its speak button), a detail-less row speaks; the query is `escapeHtml`-escaped in the "Aucun résultat" message.
